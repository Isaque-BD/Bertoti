Layered Architecture Style

Esse tipo de estrutura pode ser utilizado para um site de home banking, pois é fácil de fazer manutenções e muito fácil de aplicar as coisas, escolhendo dois requisitos não funcionais, seria o custo baixo sendo muito bom para esse tipo de arquitetura e escalabilidade sendo um requisito fraco.

Pipeline Architecture Style

Esse tipo de estrutura pode ser utilizado para um software de coleta de dados de usuários, pois esse tipo de estrutura é capaz de fazer isso, podendo fazer em tempo real. Um requisito não funcional para esse tipo de arquitetura que é ótimo é a sua grande simplicidade de programação, e um fraco seria fault tolerance.

Microkernel Architecture Style

Um bom exemplo de software para implementar esse tipo de arqutitetura é uma IDE, como por exemplo, o Eclipse, pois esse tipo de arquitetura você pode adicionar várias features facilmente, sendo muito bom para esse modelo. Sua evolucionaridade é muito boa para esse tipo de software, porém sua elasticidade é ruim.

Service-Based Architecture Style

Um exemplo de software que pode ser utilizado para esse tipo de estrutura é gerenciamento de dados de uma empresa, pois esse tipo de arquitetura tem várias partições, podendo o usuário escolher várias ferramentas que deseja, essa arquitetura tem um ótima facilitação para implementação de recursos, porém sua elasticidade é ruim.

Event-Driven Architecture Style

Um software de banco é uma boa implementação para esse tipo de arquitetura, devido a sua alta escalabilidade e oferecendo várias ferramentas para seus usuários.Seu trade off ganha bastante para escalabilidade que é muito útil para esse tipo de software, porém ele perde na testabilidade.

Microservices Architecture

Serviço de Streaming é uma implementação boa para esse tipo de arquitetura, pois facilita sua escalabilidade e ajuda no desenvolvimento de um aplicativo, para esse tipo de software é muito importante a escalabilidade, por isso essa arquitetura ganha nesse requisito, porém perde na questão de custo, que é muito alto.
